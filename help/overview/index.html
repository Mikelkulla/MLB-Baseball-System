System Architecture

How the system is structured, where every file lives, and how data flows from raw API responses to a final bet pick.

What This System Does

The MLB Betting System is a Python-based moneyline prediction engine that fetches live data from five sources, runs a multi-layer probability and EV model, and serves results through a FastAPI browser dashboard.

  • Fetches live odds, injuries, weather, sharp-money splits, and pitcher data
  • Computes vig-free probabilities adjusted for pitching, bullpen, and injuries
  • Calculates Expected Value, confidence, and tier for every game
  • Applies four safety layers before recommending a unit size
  • Serves a live dashboard at localhost:8000

End-to-End Data Flow

Phase 1 — Data Collection
odds_client → Game[] (ML, spread, total, opening lines)
injury_scraper → RawInjury[] (Covers.com)
weather_client → WeatherData{} (WeatherAPI.com)
dk_scraper → SplitEntry[] (DK Network — handle% / bets%)
pitcher_client → PitcherStats{} (MLB Stats API probable starters)
bullpen_client → BullpenStats{} (MLB Stats API team pitching)

Phase 1B — Apply Impacts
injury_engine → game.away/home_injury_impact (prob delta)
→ game.sp_gate_blocked (True if SP Out/Doubtful)
weather_engine → game.weather_over/under_adj (O/U points)
pitcher_engine → game.away/home_pitcher_score (0–100)
bullpen_engine → game.away/home_bullpen_score (0–100)
park_factors → game.park_factor, game.park_ou_adj
dk_splits → game.away/home_handle_pct, game.sharp_split_score

Phase 2 — Prediction Engine (per game)
1. Vig-free ML probability (away%, home%)
2. Injury probability adjustment
3. Pitcher probability adjustment (park-scaled, max ±10pp)
4. Bullpen probability adjustment (park-scaled, max ±4pp)
5. EV for both sides → pick the higher EV side
6. Negative EV → PASS immediately
7. CLV delta (opening vs current spread)
8. SharpSplit = ourHandle% − ourBets%
9. WPI = 50 + (ourHandle − 50) × 1.5
10. Confidence = evNorm×0.5 + SharpScore×0.5
11–13. Safety caps (steam, lineflip, WPI gates)
14. Tier assignment → Prediction

Output
Qualified picks (non-PASS) → live_predictions_YYYYMMDD.json
All games (incl. PASS) → mlb_model_YYYYMMDD.json
Both → CSV files + browser dashboard

Core Domain Objects

Game models/game.py

The primary object that flows through every stage. Created by odds_client.py and annotated by each engine module.

FieldTypeDescription
game_idstrUnique ID from The Odds API
away_team / home_teamstrCanonical team key (e.g. new_york_yankees)
oddsGameOddsML, spread, total, opening lines, consensus probs
away/home_injury_impactfloatProbability delta from injuries (negative = hurt)
weather_over/under_adjfloatO/U line adjustment points from weather
sp_gate_blockedboolTrue if probable SP is Out/Doubtful
away/home_pitcher_scorefloat0–100 pitcher quality (50 = neutral)
away/home_bullpen_scorefloat0–100 team pitching depth (50 = neutral)
park_factorfloatHome park run factor (1.00 = league average)
park_ou_adjfloatO/U adjustment from park characteristics
sharp_split_scorefloatSSS (default 50 = neutral, not 0)
away/home_handle_pctfloatDraftKings handle% per side
Important: All defaults are 50.0, not 0.0

Fields like sharp_split_score, away_pitcher_score, and away_bullpen_score default to 50.0 (neutral). Using 0.0 would penalise the confidence score when data is unavailable — that would be incorrect behavior.

Key Design Principles

1

Pick by EV, not probability

A 45% underdog at +280 can have better EV than a 55% favourite at -190. The system always picks the highest-EV side. This is the core V8.0 alignment.

2

Fielding-independent pitcher scoring

ERA and WHIP are stored for display but excluded from scoring. FIP, K/9, BB/9, and HR/9 are used because they don't depend on the defense behind the pitcher.

3

Park-scaled pitching edges

Pitcher and bullpen probability adjustments are multiplied by a park scaling factor. Coors Field (1.38) reduces the edge by 38% — great pitching matters less there. Petco Park (0.88) amplifies the edge by 12%.

4

Multiple safety layers before units are assigned

Steam-against cap, steam auto-pass, SP gate, WPI gate, and LineFlip cap all run after confidence is computed. They can reduce or zero out the recommended units without changing the tier label.
