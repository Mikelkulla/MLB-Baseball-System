Pipeline

How MLBPipeline orchestrates data collection, impact annotation, and prediction generation across three phases.

Phase 1 — Data Collection

All five data sources are fetched and stored in-memory by pipeline.py. Each fetch is independent and can be triggered individually on different schedules.

SourceMethodStored asFile
The Odds APIHTTP GETself._games: list[Game]data/odds_client.py
Covers.com injuriesHTTP scrapeself._injuries: list[RawInjury]data/injury_scraper.py
WeatherAPI.comHTTP GETself._weather: dict[str, WeatherData]data/weather_client.py
DK Network splitsHTTP GET + parseself._splits: list[SplitEntry]data/draftking_scraper.py
MLB Stats API pitchersHTTP GETself._pitchers: dict[str, PitcherStats]data/pitcher_client.py
MLB Stats API bullpensHTTP GETself._bullpens: dict[str, BullpenStats]data/bullpen_client.py

Phase 1B — Apply Impacts

The _apply_impacts() method annotates each Game object with computed values. This runs before every Phase 2 cycle so the engine always has fresh impact data.

1

Injuries

Builds a lookup team_key → [RawInjury]. Computes position-weighted probability deltas for each team. Sets game.away/home_injury_impact and game.sp_gate_blocked.

2

Weather

Looks up weather by home team key. Skips domes automatically. Calls WeatherImpactEngine.calculate() and sets game.weather_over_adj / game.weather_under_adj.

3

Pitchers

Looks up PitcherStats by team key. Sets game.away/home_pitcher_score (0–100) and game.away/home_pitcher_name. Defaults to score=50, name="TBD" if no data.

4

Bullpen

Looks up BullpenStats by team key. Sets game.away/home_bullpen_score (0–100). Defaults to 50 if team has fewer than 10 games played.

5

Park Factors

Calls get_park_factor(game.home_team) and park_ou_adjustment_display(game.home_team). Sets game.park_factor and game.park_ou_adj.

6

Sharp Splits

Matches DK split entries to games by (away_key, home_key) tuple. Sets game.sharp_split_score, away/home_handle_pct, and away/home_bets_pct. Defaults all to 50.0 if no match.

Phase 2 — Prediction Engine

Runs immediately after _apply_impacts(). Calls PredictionEngine.evaluate(game) for every game in self._games.

See the Probability, EV, and Confidence pages for the full step-by-step logic.

Output: two lists — qualified picks (non-PASS, saved to live_predictions_YYYYMMDD.json) and all predictions (every game including PASS, saved to mlb_model_YYYYMMDD.json).

Scheduler — Background Jobs

scheduler/runner.py uses APScheduler to run refresh jobs at different intervals. The scheduler starts automatically with uvicorn app:app.

Job IDDefault intervalWhat it does
full_refresh360 minAll phases in sequence
odds30 minOdds API only
injuries120 minCovers.com scrape
weather240 minWeatherAPI fetch
dk_splits180 minDK Network scrape
pitchers60 minMLB Stats API probable starters
bullpens120 minMLB Stats API team pitching
live_predictions10 minRe-run engine with cached data

Manual Refresh via API

# Full refresh (all phases)
POST /api/predictions/refresh

# Odds only
POST /api/predictions/refresh/odds

# Pitchers only
POST /api/predictions/refresh/pitchers
Refresh order matters

The engine always re-runs _apply_impacts() before generating predictions, so new data from any individual refresh job is automatically picked up on the next live_predictions cycle (every 10 min).
