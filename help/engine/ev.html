Expected Value

How EV% is calculated, what it means, why the pick is chosen by EV not probability, and when EV is shown as "—".

EV Formula

Expected Value measures the average return per unit staked across many identical bets. A positive EV% means profitable over the long run.

American Odds → Decimal
If odds ≥ 0: decimal = 1 + odds / 100
If odds < 0: decimal = 1 + 100 / |odds|

# Examples: +130 → 2.30, -150 → 1.667, -110 → 1.909
EV% Calculation (stake = 1 unit)
profit_if_win = decimal − 1
EV = (prob × profit_if_win) − ((1 − prob) × 1)
EV% = EV × 100

# Example: 55% probability at -110
decimal = 1.909, profit = 0.909
EV = (0.55 × 0.909) − (0.45 × 1) = 0.500 − 0.450 = +0.050
EV% = +5.0%

What EV% Means in Practice

EV%Meaning
+5.0%Expected to return +0.05u profit per 1u bet, long-term average
+2.0%Thin edge — qualifies but treated carefully
0.0%Break-even — market probability exactly matches model probability
−4.7%The typical EV when NO adjustments are active (spring training) — the vig%
−10%+Extreme underdog at poor price
Why both sides show the same EV in spring training

When pitcher/injury adjustments are inactive (all scores = 50), the model probability equals the market-implied probability. In this case EV = 1/overround − 1, which is the same negative value for both sides. EV only diverges between sides when the model disagrees with the market.

Odds Gate (±400)

Lines outside −400 to +400 return None from the EV calculator and display as "—" in the model table.

Reason: extreme favorites (e.g. −800) are not realistic betting targets — the required edge to achieve positive EV on a −800 line is unreachable, and the pick would be systematically misleading.

# config/settings.py
EV_ODDS_MIN = -400
EV_ODDS_MAX = +400

Pick Selection — EV, Not Probability (V8.0)

The system calculates EV for both sides independently, then picks the side with the higher EV.

Pick Selection
away_ev = EVCalculator.calculate(away_prob, away_ml_odds)
home_ev = EVCalculator.calculate(home_prob, home_ml_odds)

pick = side with higher EV
if best_ev < 0: → PASS immediately (negative EV gate)
Why EV, not probability?

A 45% underdog at +280 has better EV than a 55% favourite at −190. Picking by probability alone would always take the higher-probability side, even when the price is terrible. Picking by EV ensures you only bet when the odds offer value relative to the model's probability estimate.

Negative EV Gate

If the best EV across both sides is negative, the game is immediately classified as PASS — no further processing. This is the single most important filter. During spring training when no adjustments are active, every game will fail this gate.
