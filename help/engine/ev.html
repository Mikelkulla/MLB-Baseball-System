Expected Value

How EV% is calculated, what it means, why the pick is chosen by EV not probability, and when EV is shown as "—".

Probability Source — Always Moneyline

Win probability is always derived from ML (moneyline) odds — never from run line prices.

Why not the run line?

The run line is a different event: it requires winning by 2+ runs. Since ~28% of MLB games end by exactly 1 run, a team's run line win probability is roughly ML_win_prob × 72%. If you use run line prices for both probability and EV, the two cancel out and EV always equals −(vig%) — every game PASSes regardless of the model's adjustments.

EV is calculated against the best available ML price across all books. Probability and EV target the same event (win the game), so the comparison is correct.

EV Formula

Expected Value measures the average return per unit staked across many identical bets. A positive EV% means profitable over the long run.

American Odds → Decimal
If odds ≥ 0: decimal = 1 + odds / 100
If odds < 0: decimal = 1 + 100 / |odds|

# Examples: +130 → 2.30, -150 → 1.667, +100 → 2.00
EV% Calculation (stake = 1 unit)
profit_if_win = decimal − 1
EV = (prob × profit_if_win) − ((1 − prob) × 1)
EV% = EV × 100

# Example: consensus ML gives 55% true probability, best book offers +120
decimal = 2.20, profit = 1.20
EV = (0.55 × 1.20) − (0.45 × 1) = 0.660 − 0.450 = +0.210
EV% = +21.0% ✓ PICK

What EV% Means in Practice

EV%Meaning
+10%+Strong edge — consensus probability vs. single book ML divergence
+2–5%Thin edge with adjustments — pitcher/injury signal pushing probability
0.0%Break-even — model probability exactly matches the book's implied probability
−4.7%Typical EV when no edge exists — the vig embedded in the odds
−10%+Underdog at poor price, or model strongly disagrees in the wrong direction

Odds Gate (±400)

Lines outside −400 to +400 return None from the EV calculator and display as "—" in the model table.

Reason: extreme favorites (e.g. −800) are not realistic betting targets — the required edge to achieve positive EV on a −800 line is unreachable, and the pick would be systematically misleading.

# config/settings.py
EV_ODDS_MIN = -400
EV_ODDS_MAX = +400

Pick Selection — EV, Not Probability

The system calculates EV for both sides independently, then picks the side with the higher EV.

Pick Selection
away_ev = EVCalculator.calculate(away_prob, best_away_ml_price)
home_ev = EVCalculator.calculate(home_prob, best_home_ml_price)

pick = side with higher EV
tiebreaker = underdog (positive ML odds)
if best_ev < 0: → PASS (units = 0, tier kept for display)
Why EV, not probability?

A 45% underdog at +280 has better EV than a 55% favourite at −190. Picking by probability alone would always take the higher-probability side, even when the price is poor. Picking by EV ensures you only bet when the odds offer value relative to the model's probability estimate.

Soft EV Gate

When the best EV across both sides is negative, the system applies a soft gate based on the model's win probability for the picked side:

if EV < 0 AND prob ≥ 55% → pick kept, units capped at 1.0u # "EV-CAP"
if EV < 0 AND prob < 55% → PASS (0 units)

This means the system can still surface picks during periods when EV is structurally negative (e.g. spring training, when no independent probability edge exists), as long as the probability signal is meaningful. The tier label and confidence score are always shown so you can evaluate the model's view regardless of the gate result.

Why 55%?

55% matches the GOLD tier minimum confidence threshold — it is the lowest level at which the model considers a probability signal non-trivial. Below 55% the model has no meaningful edge over a coin flip at −110 prices, so a hard PASS is applied.
