Bullpen Scoring

Team pitching depth scored 0–100, its proportional probability shift, and why the max adjustment is ±4pp vs ±10pp for starters.

What the Bullpen Score Represents

The bullpen score is a team-aggregate pitching proxy using the MLB Stats API's season pitching stats for the entire staff (starters + relievers). It correlates with overall bullpen quality even though the probable starter is already scored separately.

Acknowledged approximation

The MLB Stats API returns whole-staff aggregate stats, not pure reliever stats. In practice, aggregate FIP/K9/BB9 correlates well with true bullpen quality because the probable starter is scored in the pitcher engine — the remaining signal in the aggregate reflects reliever quality.

Scoring Formula

Uses the same three FIP-based metrics as the pitcher engine (HR/9 excluded — too noisy for bullpen aggregates).

Bullpen Composite Score
fip_score = (league_avg×2 − FIP) / (league_avg×2) × 100 # lower FIP = higher score
k9_score = K/9 / (league_avg×2) × 100 # higher K/9 = higher score
bb9_score = (league_avg×2 − BB/9) / (league_avg×2) × 100 # lower BB/9 = higher score

score = fip_score×0.45 + k9_score×0.30 + bb9_score×0.25
# Clamped to [0, 100]

Spring Training Gate

Teams with fewer than 10 games played in the current season return a score of 50.0 (neutral). No adjustment fires. This prevents garbage pre-season samples from influencing probability.

Probability Shift

Starters pitch ~60% of innings; bullpens pitch ~40%. The max bullpen shift (±4pp) is proportional to ±10pp for starters.

Bullpen Probability Adjustment
edge = away_bullpen_score − home_bullpen_score
prob_shift = (edge / 50.0) × 2.0 × park_scaling # max ±4pp base

# Example: away BP score 70, home BP score 45, park neutral
edge = +25, prob_shift = (25/50) × 2.0 × 1.0 = +1.0pp toward away

Park scaling: same as pitcher — Coors (0.62) reduces the max to ±2.5pp; Petco (1.12) increases it to ±4.5pp.

Data Source

Fetched from the MLB Stats API season stats endpoint. Refreshed every 120 minutes by default.

# Endpoint used:
GET /api/v1/teams/{mlb_team_id}/stats?stats=season&group=pitching&season=YYYY

# FIP is recalculated from raw counts:
# homeruns, baseOnBalls, hitByPitch, strikeOuts, inningsPitched
