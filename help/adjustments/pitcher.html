Pitcher Scoring

How probable starters are scored 0–100 using fielding-independent metrics, and how that score shifts win probability.

Why FIP, Not ERA

ERA is contaminated by the defense behind the pitcher — a pitcher with a poor infield will appear worse than he is. FIP (Fielding Independent Pitching) isolates only what the pitcher controls: strikeouts, walks, hit-by-pitches, and home runs.

MetricUsed in scoring?Why
FIPYes — 40%Best single predictor of future ERA (FanGraphs 2024)
K/9Yes — 25%Most durable, talent-consistent pitcher skill year-to-year
BB/9Yes — 20%Highly consistent year-to-year; reflects command
HR/9Yes — 10%Real skill signal, but partly park/luck noise
Recent ERAYes — 5%Last-3-starts form signal; small sample, low weight
ERANo — display onlyDefense-contaminated
WHIPNo — display onlySame issue: H/(BB+H) inflates based on defensive quality

FIP Calculation

FIP Formula
FIP = ((13 × HR) + (3 × (BB + HBP)) − (2 × K)) / IP + 3.17

# Constant 3.17 calibrates FIP to the same scale as ERA
# League average FIP ≈ 4.20 (2024)

The raw counts (HR, BB, HBP, K, IP) come from the MLB Stats API season stats endpoint.

Scoring Formula (0–100)

Each stat is scored independently on a 0–100 scale where 50 = league average. Scores are then combined with the weights above.

Individual Stat Scoring
# FIP (lower = better; league avg 4.20)
fip_score = (league_avg×2 − FIP) / (league_avg×2) × 100

# K/9 (higher = better; league avg 8.80)
k9_score = K/9 / (league_avg×2) × 100

# BB/9 (lower = better; league avg 3.10)
bb9_score = (league_avg×2 − BB/9) / (league_avg×2) × 100

# HR/9 (lower = better; league avg 1.15)
hr9_score = (league_avg×2 − HR/9) / (league_avg×2) × 100

# All clamped to [0, 100]
Composite Score
score = fip_score×0.40 + k9_score×0.25 + bb9_score×0.20 + hr9_score×0.10 + recent_era_score×0.05
# Clamped to [0, 100]

How Score Shifts Win Probability

The pitcher score enters apply_pitcher_adjustment() in engine/probability.py.

Probability Shift
edge = away_score − home_score # −100 to +100
prob_shift = (edge / 50.0) × 5.0 × park_scaling # max ±10pp

# Example: away score 80, home score 40
edge = +40, prob_shift = +4.0pp × park_scaling

Park Scaling

The pitcher advantage is scaled down in hitter parks and up in pitcher parks.

Spring Training Gate

During spring training and early pre-season, the MLB Stats API returns no current-season stats. All pitchers default to score = 50 (neutral). This means:

  • edge = 0 → no probability shift → no EV divergence → all games PASS
  • This is correct — real stats become available in April

Pitcher O/U Adjustment

Separate from the moneyline adjustment, pitcher_ou_adjustment() computes an additional O/U signal based on the quality of both starters combined.

Pitcher O/U Adjustment
avg_score = (away_score + home_score) / 2
edge = avg_score − 50.0
adj = −(edge / 50.0) × 2.5 # negative = Under; max −2.5 pts when both are elite
