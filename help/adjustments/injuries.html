Injury Impact

Position weights calibrated to FanGraphs WAR analytics, injury status multipliers, diminishing returns, and the SP gate.

How the Delta is Calculated

Per-Player Delta
delta = position_weight × status_multiplier
Team Impact (with diminishing returns)
For each injured player (sorted by severity):
  factor = DIMINISHING_FACTORS[count_at_same_position]
  contribution = delta × factor
  total += contribution

team_impact = MIN(total, 12.0) # max 12pp per team
# Result stored as negative (e.g. −4.5 means team is hurt)

Position Weights

Calibrated against FanGraphs positional adjustments (runs per 162 games) and MLB WAR research. Higher weight = more win probability impact when injured.

PositionWeightRationale
SP (Starting Pitcher)8.0Game-defining. Market shows 10–15pp shift for ace scratch. 8.0 is average-case (the SP gate handles extreme Out/Doubtful separately).
C (Catcher)3.0Highest positional adjustment (+12.5 runs/162g). Framing value (30–50 runs/season for elite framers) is NOT captured in FIP-based pitcher scoring.
SS (Shortstop)2.0Second hardest infield position (+7.5 runs/162g). Replacement SS scarcity is well-documented.
CP (Closer)2.0High-leverage index ~1.8. Chaining effect limits net loss to 1.0–1.5 WAR (setup man shifts up).
RP (Relief Pitcher)1.58th-inning / high-leverage role. LI 1.2–1.5.
CF (Center Field)1.5+2.5 run positional adjustment. Harder than corner OF — defensive scarcity matters.
3B (Third Base)1.2+2.5 run positional adj (same tier as 2B analytically).
MR (Middle Reliever)1.2High-leverage bridge to closer.
2B (Second Base)1.0+2.5 run positional adj.
1B (First Base)0.8−12.5 run adj. Easiest infield position to fill.
LF / RF (Corner OF)0.8−7.5 run adj. Corner OF easier to replace.
DH (Designated Hitter)0.6−17.5 run adj. No defensive scarcity. Backup hitter always available.

SP Gate

The SP Gate is a separate, hard block — it doesn't just shift probability, it zeroes units entirely when triggered.

SP Gate Trigger
if probable_starter.status in {"out", "out for season", "doubtful"}:
    sp_gate_blocked = True → safe_units = 0

Rationale: an ace scratched late (Out/Doubtful) causes the market to move 10–15pp in some cases. Any pre-game analysis based on the original starter is invalidated. The model cannot reliably re-price the game without knowing the replacement pitcher.

SP Gate does not change the tier

The tier label (ELITE, STRONGEST, etc.) is preserved, but safe_units = 0. The model still shows the underlying analysis — it just won't recommend a bet. You'll see the pick in the MLB Model table but it won't appear in Live Picks.
